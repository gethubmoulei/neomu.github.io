我是一名嵌入式软件工程师，长期关注汽车电子、工业控制、CAN 总线协议与自动化测试平台。当前主要负责摩托车 BCM、通机 ECU 及相关测试设备的软件研发，工作覆盖需求分析、软件架构设计、底层驱动、应用逻辑、测试验证和现场技术支持。

## 个人优势

拥有 5 年以上工业控制与嵌入式系统开发经验，熟悉从底层驱动到应用层逻辑的完整开发流程。长期使用 PowerPC 架构 SPC56 系列及国产兼容 MCU，理解启动流程、异常处理、中断机制、Flash / SRAM 管理与硬件调试。

熟悉 CAN 总线及 UDS、CCP、J1939、K-Line 等汽车电子相关协议，能够围绕量产项目完成诊断、通信、Bootloader、Flash 模拟 EEPROM、休眠唤醒和定时器任务调度等模块开发。

除嵌入式 C 开发外，也使用 Python、PyQt5、MongoDB 与 Web 技术构建生产测试、老化测试和测试数据管理系统，提升测试效率、数据追溯能力和工程协作体验。

## 技术栈

- 嵌入式开发：C、SPC56、STM32、Bootloader、RTOS、Flash / SRAM 管理
- 汽车电子：CAN、UDS、CCP、J1939、K-Line、SPI、I2C、RS-232/485
- 测试与工具：Python、PyQt5、JSON 配置化测试、MongoDB、Web 查询系统
- 工业控制：Modbus RTU、PLC、HMI、DCS 对接、控制柜设计与现场调试
- 工程协作：Gitea、版本管理、问题定位、根因分析
